Paginator

Introduction

Sometimes we need a way to paginate our content. LazyLoad could be no convenient on some cases to reachs that even we might think on apply both techniques.

Here is an easy wayto paginate our content easily with a non endless posibilities to apply to our websites without be aware about how to structure the content.

Just setting up fastly on each of your sections what you want to be paged, (yes, you can use more than one on the same page), either through the tags in html or calling explictly on a JS script, you will have your content well structured responsivness and prepared for any resolution or any resize on the page.

Usage Manual

Basic usage

The basic component is pretty easy to use initially, just need seconds to have it ready. There are some other different configurations available to setting up which needs a bit more of work, but it's not a big deal to manage with it and with the following guide you'll get it as easy as child's play.

As the example below, just insert the html div tag in your targeted container.

<html>
  <head>
    <script type="text/javascript"  src="js/paginator/paginator.js"></script>
    <script type="text/javascript"  src="js/paginator/page.js"></script>
  </head>
  <body>
    <!-- Other content on body -->
    <div>
      <!-- Content to be paginated or other stuff -->
      <!-- Paginator cont -->
        <div class="c-paginator style-1"></div>
      <!-- -->
    </div>
  </body>
</html>

And this will produce this:

Follow the instructions below to setting up the component.
You have also an extended demo page where take examples as reference.

Component usage

They're two ways to setting up your paginator component. This ways could be mixed between on the same page.

A guide of styles and parameters to setting up the component on each case is included those links within.

Only HTML component

 <div class="c-paginator [style-N]" data-params="{[option1:value1;option2:value2...]}"></div>

The component will be created once the HTML doucment will be entirely loaded besides the paginator.js script.

This options don't allow you to control the paginator after it had have been created.
It's suitable to simple scenarios where, in example, the number of pages won't are change.

HTML & Javascript component

HTML 
 
<!-- Here the posible data-options will be overwritten by the JS parameters -->
<div id="paginator-1" class="c-paginator [style-N]" data-isAsync="true"></div>
Javascript 
//Use the jquery selector
var paginator1 = $("#paginator-1").paginator({[option1:value1;option2:value2...]})

Parameters

There is the list of parameters to be included on the html attribute or to be used in the Javascript contructor first and unique argument:

data-isAsync

NameScopeTypeDefaultDescription
data-isAsync HTML boolean false Determines if the c-paginator will start inmediatly or wait to be started by JS after.

data-params | params

NameScopeTypeDefaultDescription
data-params | params HTML | JS object --- Setting up the parameters of the paginator
OptionTypeDefaultDescription
nPages int 20 Number of pages of the paginator
* This parameter could be overrided by nItemsPage and nItemsTotal.
nPagesShown int 7 Number of pages shown of the paginator.
nItemsPage int 5 Number of items whose compose each page.
* This parameter beside nItemsTotal override the nPages paramteer.
nItemsTotal int 100 Number of items whose compose each page.
* This parameter beside nItemsPage override the nPages parameter.
settingsTheme object --- Object containing properties related to the behavior of the paginator.
allResponsive boolean false Put the controls on the same container as page links
triggerEvent boolean true Paginator triggers "onPageChange" event.
collapseAllCntrl boolean true Collaspse controls when they're not necessary.

API

Here is the available methods to be used when the initialization has been done through JS:

Constructor

Init the paginator with the corresponding paramsand return the instance of the object.

$("#paginator-1").paginator(params);
OptionTypeDefaultDescription
params object {} Parameters to init the paginator. See

goLink

Force externally the paginator to go to the desired page.

$("#paginator-1").paginator("goLink", indexLink, *indexOldLink);
OptionTypeDefaultDescription
"goLink" string "goLink" Indicates explictly the method who goes to be called.
indexLink int Index of the new page.
*indexOldLink int undefined Refers the index of the old page. (The performance will be fast)

remove

Remove the paginator instance and the content of the container within.

$("#paginator-1").paginator("remove");
OptionTypeDefaultDescription
"remove" string "remove" Indicates explictly the method who goes to be called.

Events

All the events will bre triggered from the DOM element who inits the paginator.

EventDescriptionData
change Triggered when a new page has been selected. No.
onPageChange Triggered when a new page has been selected. But retrieve the newPage on data.
This event is disabled when params.settingsTheme.triggerEvent is false.

newPage: The index of the new page selected.
